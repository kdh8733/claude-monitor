# claude-monitor

Claude Max 구독의 **한도 소진율**과 **사용 귀속**을 지속 기록하는 개인용 대시보드.

## 왜 만드는가

Claude Max 구독자에게는 토큰당 요금이 존재하지 않는다. 청구액은 매달 고정이고,
추가 사용량 과금을 꺼두면 한도를 넘어도 돈이 나가는 게 아니라 **작업이 막힌다.**
그래서 희소한 자원은 돈이 아니라 **한도 헤드룸**이다.

그런데 그 헤드룸을 얼마나 남기고 버리는지 아무도 모른다. 이유는 두 가지다.

1. Anthropic은 **현재 시점 게이지만** 준다. 히스토리가 없다.
   지금 주간 한도의 41%를 썼다는 건 알 수 있어도, 지난 6주간 평균 몇 %를 버렸는지는 알 수 없다.
2. Claude Code 트랜스크립트는 **보존 기간이 지나면 사라진다.**
   "어느 프로젝트가 한도를 먹었나"를 나중에 물어볼 수 없다.

기록을 시작하는 것 자체가 이 프로젝트의 1차 가치다.

## 두 데이터 소스

| | `/api/oauth/usage` | 트랜스크립트 JSONL |
|---|---|---|
| 답하는 질문 | 탱크가 얼마나 찼나 | 어디에 썼나 |
| 권위 | 있음 (claude.ai 웹 사용분 포함) | 없음 (Claude Code 사용분만) |
| 히스토리 | 없음 (스냅샷) | 있음 (단, 소멸함) |
| 귀속 | 없음 | 프로젝트/모델/세션/서브에이전트 |

두 소스는 대체재가 아니라 **보완재**다. 게이지는 잔량을, 트랜스크립트는 귀속을 답한다.
어느 한쪽만으로는 "요금제를 잘 쓰고 있는가"에 답할 수 없다.

> 트랜스크립트의 롤링 토큰 합으로 한도 소진율을 역산하려는 시도는 실패했다.
> 5시간 누적 309K 토큰에서 한도에 걸린 날이 있고, 13.0M을 쓰고도 걸리지 않은 날이 있다.
> Claude Code 밖(claude.ai 웹/데스크톱)의 사용이 같은 한도를 소모하기 때문이다.
> 근거는 [`ROADMAP.md`](./ROADMAP.md) 의 "반증된 가설" 절에 있다.

## 스택

TypeScript 단일 언어. 런타임 의존성 최소.

- 수집기: Node 24 (폴링 데몬 + refreshToken 갱신 + 소스 어댑터)
- 저장: `node:sqlite` (Node 내장, WAL). 원본 raw JSON 보존 + 파생 집계
- 웹: Next.js App Router (로컬은 RSC가 읽기 전용으로 조회, 데모는 정적 export)
- 차트: visx / UI: shadcn/ui + Tailwind v4

선정 근거와 기각된 후보는 [`docs/decisions/001-stack.md`](./docs/decisions/001-stack.md) 참조.

## 데이터 취급

이 저장소는 **코드만** 공개한다.

트랜스크립트에는 프롬프트 원문과 실제 프로젝트명이 들어 있다.
`data/`, `*.sqlite`, `snapshots/` 는 `.gitignore` 로 차단돼 있으며 **이 항목은 완화 대상이 아니다.**
스크린샷과 데모는 `fixtures/demo/` 의 합성 시드로만 만든다.

자격증명(`~/.claude/.credentials.json` 의 OAuth 토큰)은 로그, 에러 메시지,
테스트 픽스처, 커밋 어디에도 남기지 않는다.
